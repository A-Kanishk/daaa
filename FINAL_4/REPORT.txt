2. Introduction
Betweenness centrality is a widely used measure in network analysis because it quantifies
how frequently a vertex lies on shortest paths between other vertex pairs. Vertices with
high betweenness often act as structural bridges that control information flow between
communities.
Computing exact betweenness at scale is expensive. Naive all-pairs strategies become
impractical on large graphs, while Brandes’ algorithm provides a major improvement
by combining shortest-path counting with dependency accumulation in a single-source
framework. For unweighted graphs, this yields O(nm) time and O(n + m) space.
In this work, we implement and evaluate Brandes-based computation on three SNAP
datasets under the assignment constraint that all graphs are treated as undirected and
unweighted. Exact runs are reported for wiki-Vote and email-Enron; sampled-source
approximation is used for as-Skitter due day-scale exact runtime on available hardware.

3. Algorithm Summary
3.1. Reference
Ulrik Brandes, A faster algorithm for betweenness centrality, Journal of Mathematical
Sociology, 25(2):163–177, 2001.

3.2. Core idea
For each source vertex s, the algorithm performs:
1. A BFS traversal (unweighted case) to compute shortest-path counts σs (·), distances,
and predecessor lists.
2. A reverse dependency accumulation phase to update centrality values.
For unweighted graphs, complexity is:
T = O(nm),

S = O(n + m)

where n is number of vertices and m is number of edges.

3.3. Two-phase dependency accumulation
For each source s, the computation has two phases:
1. Forward BFS phase: compute distance d(·), shortest-path counts σ(·), and predecessor lists P (·).
2. Reverse accumulation phase: process vertices in reverse distance order and propagate dependencies using:

σs (v)

The final betweenness score is obtained by summing dependencies over all sources; for
undirected graphs, values are divided by 2 to avoid double counting.

3.4. What is implemented from the paper
The implementation follows Brandes’ exact dependency-accumulation method for betweenness centrality:
1. Single-source shortest paths from each source vertex using BFS (unweighted case).
2. Computation of shortest-path counts σ and predecessor lists P during traversal.
3. Reverse accumulation of dependencies δ to update centrality values.
4. Final division by 2 for undirected graphs (as each pair is counted twice).

3.5. Approximation for very large graphs
For as-Skitter, we additionally use sampled-source approximation (Bader et al., 2007):
instead of running all n sources, we run k randomly sampled sources and scale scores by
n/k. This changes practical runtime from O(nm) to O(km) while preserving high-quality
top-k rankings in practice.

4. Dataset Descriptions
All datasets are from the SNAP collection and are treated as undirected and unweighted.

4.1. wiki-Vote
Wikipedia voting network where nodes represent users and edges represent voting interactions. After preprocessing, the graph has 7,115 vertices and 100,762 undirected edges.

4.2. email-Enron
Email communication network inside Enron. Nodes are email addresses and edges represent communication links. After deduplication and undirected conversion, the graph has
36,692 vertices and 183,831 edges.

4.3. as-Skitter
Internet topology graph collected by CAIDA skitter. Nodes represent autonomous systems and edges represent observed connectivity. This is the largest graph with 1,696,415
vertices and 11,095,298 edges, and is the only dataset evaluated with sampled-source
approximation.

5.1. Language and execution
• Language: C++17

5.2. Key implementation features

• Uses adjacency-list representation for linear-space graph storage.
• Handles SNAP edge-list comments and parsing robustly.
• Uses undirected treatment for assignment runs (default mode).
• Reports execution time and memory usage (estimated + measured peak RSS).

8. Observations and Discussion
1. Scalability behavior: Exact Brandes is practical for small-to-medium graphs (wikiVote, email-Enron), while sampled-source mode is necessary for as-Skitter under
realistic deadlines.
2. Space efficiency: Observed memory usage remains linear in graph size (O(n + m)),
which is the key advantage over naive all-pairs formulations.
3. Ranking stability under sampling: Across sampled runs, top-ranked as-Skitter
nodes remain stable, supporting approximation quality for top-20 analysis.
4. Network structure insight: In all datasets, betweenness is highly skewed: a small
set of vertices dominates shortest-path mediation.
